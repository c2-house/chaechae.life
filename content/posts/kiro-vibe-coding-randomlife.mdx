### 목차

## 한국에서 다시 태어나기 vs 랜덤 돌리기

열심히 살고 있지만 살아 가는게 참 쉽지 않습니다. 많은 사람들이 그렇게 느끼고 있는 것 같습니다. 최근에 친구랑 대화하다가 친구가 **다시 태어날 수 있으면 한국에서 태어나기 vs 랜덤 돌리기,** 중에 어떤 걸 선택할래? 라고 물어봐서 재밌게 이야기 했었습니다. 더 좋은 환경에서 태어날 수도 있으니 랜덤을 돌려보겠다고 했다가 맹비난을 받았는데요. 그 얘기를 듣다가 이걸 웹 사이트로 만들어 보면 재밌겠다 싶어서 KIRO를 이용해서 만들어봤습니다.

간단한 웹 사이트를 만들 때 KIRO의 Spec이 정말 활용도가 좋았습니다. 이 웹사이트는 요구사항이 정말 간단합니다.

- 다시 태어나기 버튼을 누르면 새로운 나라가 선택됨
- 다시 태어나는 나라별 확률은 국가의 인구수에 비례함
- 다시 태어나는 나라가 선택되면 해당 국가가 지도에 표시됨

요구사항을 입력하고 어떤 기술로 개발할지도 명시해줬습니다.

- Vite + Typescript + React
- Component based Architecture

프롬프트에 이런 내용만 담아서 개발해달라고 요청하니 알아서 위 문서들을 생성하고 검토를 요청합니다. 처음 KIRO를 사용해봤을 때 **속도가 왜 이렇게 느릴까? 라고 생각했었는데 이제 그 부분도 많이 개선된 것 같습니다. 엄청 빠르지는 않지만 속도가 어느 정도 올라온 것 같습니다.**

핵심 기능은 몇 분만에 금방 완성이 되었습니다. 세션이 길어지면 중간에 끊기는데 세션을 요약해서 새로운 세션으로 이어갈 수 있는 기능이 있어서 좋았습니다.

## KIRO Vibe로 추가 기능 개발하기

금방 만들어서 친구들에게 보여주고 테스트 해보니 일반적으로 한국 보다 상황이 좋은 나라는 잘 선택되지 않았습니다. 그러다가 문득 ‘일반적으로 후진국이나 개발 도상국이라고 해서 해당 국가의 모든 사람의 삶의 질이 별로라고 할 수 있나?’ 라는 생각이 들었습니다. 반대도 마찬가지구요. 그래서 **새로운 인생에도 랜덤한 기능**을 추가하고 싶어졌고 **새로운 인생을 AI로 시뮬레이션** 해보면 좋겠다는 생각이 들었습니다.

KIRO의 Vibe 세션을 새로 열어서 위 기능을 설명하고 만들어 달라고 요청했습니다. **알아서 작성된 코드를 읽고 전체 컨텍스트를 파악한 뒤에 개발**을 시작해서 뚝딱 만들었습니다.

사실 전 세계 국가의 정보를 제공하기 위해서 CIA에서 제공하는 국가 데이터를 크롤링해서 전달해주긴 했습니다. 대화 몇 번 만에 완성이 되어 금방 배포했습니다. 또 하나 좋았던 점은 AI가 개발할 때 빌드가 잘 되는지도 스스로 테스트 해보는 것이 었습니다. **빌드해보라고 시키지도 않았는데 하더라구요.**

## 마무리

이 프로젝트를 개발하고 힘들어도 현생에 감사하면서 살아야겠다는 생각을 많이 했습니다. 그리고 KIRO는 preview 기간에 많이 사용해보면서 계속 쓰고 싶다는 생각이 들었습니다. **개인적인 DX는 좋았어요.** 최근에는 아주 복잡한 것은 개발하고 있지 않아서 그런걸 수도 있겠다는 생각이 들기는 했습니다. 그리고 `바이브코딩` 이라는 것이 해보면 해볼 수록 개발에 대해서 잘 알 수록, 더 잘 되고 쉬워서 비개발자라면 개발을 좀 배우고 해보면 좋겠다라는 생각이 계속 듭니다.
