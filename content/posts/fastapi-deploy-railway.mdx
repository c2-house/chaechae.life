### 목차

## FastAPI 배포하는 방법

FastAPI를 이용해서 어떤 서버를 개발하고 나면 배포를 해야합니다. 간단한 토이 프로젝트라도 배포를 해보느냐 해보지 않느냐는 경험치 측면에서 차이가 발생하죠. FastAPI 서버를 배포하는 방법은 다양합니다. 회사에서 일 할때는 **AWS를 많이 이용**했었는데(물론 지금도 씁니다만), 지금은 교육 쪽에 있어서 개발은 개인 프로젝트를 많이 하다보니 AWS같은 서비스는 잘 사용하지 않게 되더라구요. 딸깍으로 배포하는 여러 서비스들이 있습니다. **Render**도 자주 이용하지만 이번 포스팅에서는 **Railway**라는 서비스에 대해서 다뤄보려고 합니다. 이것도 딸깍으로 아주 쉽게 배포할 수 있습니다.

## 프로젝트 생성 방법

프로젝트를 생성하기 위해서 콘솔에서 진행할 수도 있고 Railway에서 제공하는 CLI를 통해 진행할 수도 있습니다.

콘솔에서 생성하는 방법은 로그인 후 나오는 `New project`에서 순차적으로 진행하면 됩니다. 화면에서 나오는 대로 진행하면 되는 것이라서 어려울 것이 없습니다. 가장 기본적인 방법은 Github에 프로젝트를 저장하고 그 Repo와 연동하는 방법입니다.

```
npm i -g @railway/cli
```

## CLI로 배포하기

Railway를 써보고 정말 좋았던 점이 **CLI로 배포하기가 정말 쉽다**는 것입니다. 다음과 같은 구조를 가진 FastAPI프로젝트가 있습니다.

<Image
  alt="FastAPI 프로젝트 구조"
  caption="FastAPI 프로젝트 구조"
  src="/fastapi-deploy-railway/02.png"
  fullWidth={false}
/>

`railway.json` 파일을 하나 만들어 줍니다. 아니면 `railway.toml`로 만들어도 되는데 문법이 조금 다르겠죠?

`railway.json`에는 빌드할 때, 배포할 때 필요한 값들을 넣어줘야 합니다. 대표적으로 시작 명령이 있습니다.

```json
// railway.json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "uvicorn app.main:app --host 0.0.0.0 --port $PORT"
  }
}
```

`railway.toml`을 쓴다면 다음과 같습니다.

```toml
# railway.toml
[build]
builder = 'nixpacks'

[deploy]
startCommand = "uvicorn app.main:app --host 0.0.0.0 --port $PORT"
```

만약 배포를 Docker로 한다면 위 파일은 필요없습니다. 너무 당연하지만 대신 `Dockerfile` 이 있어야 합니다.

파일 준비가 끝났으면 다음과 같은 명령을 입력해야 합니다.

```bash
# 로그인
railway login

# 프로젝트 생성
railway init

# 배포
railway up

# 프로젝트를 이미 생성한 경우
railway service
```

로그인하고 프로젝트 생성하고 배포하면 끝입니다. 만약 콘솔에서 먼저 프로젝트를 생성했다면 `service` 명령을 입력하면 되는데 **미리 생성된 프로젝트와 연결**할 수 있습니다. 배포가 아래 이미지 처럼하면 끝납니다.

<Image alt="Railway CLI 배포" caption="Railway CLI 배포" src="/fastapi-deploy-railway/03.png" />

`railway up` 을 할 때 `.gitignore` 에 있는 파일들은 무시가 됩니다. `.env` 무시되어 환경변수가 없어서 배포에 실패했는데요. Settings에서 `Shared Variables` 를 이용하거나 `Docker` 이미지로 배포하거나 배포할 때 `.gitignore`에서 `.env` 를 잠시 제외하는 아주 간단한 방법도 있습니다.

<Image
  alt="Railway CLI 배포 실패"
  caption="Railway CLI 배포 실패"
  src="/fastapi-deploy-railway/04.png"
/>

배포가 끝나고나서 콘솔에 들어가보면 이미지와 같이 배포된 프로젝트를 볼 수 있습니다. 그리고 아래 이미지에서 보이는 것처럼 `Generate Domain`을 누르면 퍼블릭으로 접속할 수 있는 도메인이 생성됩니다. 아니면 `railway domain` 명령으로 CLI에서 할 수도 있습니다.

## Railway 아쉬운 점

배포가 너무나도 쉽지만 한 가지 아쉬운 점이 있는데요. **배포 Region에 서울이 없습니다. 기본 배포는 US로 잡힙니다. 하지만 Asia 싱가폴 Region이 있는데 싱가폴로 배포하면 지연이 그렇게 체감되진 않는 것 같습니다.** 그래도 사용자가 많은 서비스를 배포하긴 어렵지 않나 생각이 듭니다. **Pro plan을 이용하면 Multi-region으로 배포하는 것도 가능**합니다.

Settings에서 아래로 쭉 내리다보면 Region을 바꾸는게 있습니다.

## Railway Pricing

Railway는 완전 무료 서비스는 아닙니다만 가입하면 짧은 무료 이용기간을 줍니다. 부트캠프에서 프로젝트를 하거나 개인 포트폴리오를 잠깐 배포하는 용도로 쓴다면 괜찮지 않나 싶고, 취미용도 월 5달러 밖에 안하기 때문에 가격이 부담스럽지는 않은 것 같습니다.

<Image alt="Railway Pricing" caption="Railway Pricing" src="/fastapi-deploy-railway/07.png" />
