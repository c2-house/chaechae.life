### 목차

## 네이버클라우드 정적 웹 호스팅

웹 프론트엔드를 배포하는 여러가지 방법 중에 **오브젝트 스토리지를 이용한 정적 웹 호스팅** 방법이 있습니다. AWS를 이용해보셨다면 `AWS S3`를 이용한 정적 호스팅을 들어보셨을거라고 생각합니다. 네이버클라우드에도 비슷하게 `Object Storage`라는 서비스를 이용해서 정적 웹 호스팅을 할 수 있습니다.

S3를 이용한 배포는 `CloudFront`라는 CDN 서비스와 연결하고 도메인 연결까지해서 사용할 수 있습니다만, **제가 시도해 본 바로**는 네이버클라우드에서는 CDN이 있긴 하지만 도메인 연결에 제약이 좀 있습니다. 그래서 아직은 `Object Storage`를 이용한 정적 웹 호스팅은 데모용으로만 적합하지 않나 싶습니다.

## Object Storage 버킷 생성하기

우선 너무 당연하게도 `Object Storage`에 버킷을 생성해야합니다. 버킷 생성에는 다음과 같은 4단계가 있습니다.

<Image
  alt="Object Storage 버킷 생성 프로세스"
  src="/ncloud-object-storage-static-web-deployment/01.png"
/>

**기본 정보에서 버킷 이름을 설정**하고, **설정 관리는 그냥 넘어갑니다.** 혹시라도 ‘보안을 위해 암호화를 해야지’라고 생각해서 암호화 설정을 한다면 정적 웹호스팅은 할 수 없습니다. **권한 관리에서 전체 공개를 공개로 설정**해야합니다. 전체 공개로 해야 일반 유저가 접속할 수 있겠죠.
