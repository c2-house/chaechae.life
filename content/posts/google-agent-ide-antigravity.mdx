### 목차

## 구글이 만든 IDE Antigravity는 무엇인가

작년 여름 구글에서 만든 IDX라는 웹기반 IDE가 나왔었습니다. 지금은 Firebase Studio가 되었는데요. 이번에 구글이 또 새로운 IDE를 발표했습니다.

오늘은 구글이 만든 Agent First IDE, Antigravity에 대해서 간단하게 알아보려고 합니다. 우리가 흔히 알고 있고 사용하고 있는 VSCode나 IntelliJ 같은 기존의 IDE는 개발자가 코드를 작성하면 도구가 문법을 검사하거나 자동 완성을 도와주는 **보조적 역할**에 머물렀습니다. 하지만 **Antigravity**는 패러다임을 완전히 바꿨습니다. 바로 `Agent-first` 철학을 바탕으로 설계되었기 때문입니다.

**그렇다면** `Agent-first`**란 무엇일까요?** 기존의 AI 코딩 툴은(GitHub Copilot 등)가 운전자의 옆에서 길을 알려주는 내비게이션이라면, Antigravity의 에이전트는 직접 운전대를 잡고 목적지까지 운전해 주는 자율주행 기사에 가깝습니다.

Antigravity는 단순히 코드 한 줄을 추천하는 것을 넘어, 사용자가 "이런 기능이 있는 웹사이트를 만들어줘"라고 자연어로 명령하면, 프로젝트 전체 구조를 설계하고, 필요한 파일을 생성하고, 디자인하고 개발까지 스스로 다 합니다.

구글의 강력한 LLM(거대언어모델)인 Gemini가 IDE에 탑재되어(Claude도 이용할 수 있긴 합니다.), 개발의 중력(어려움과 진입장벽)을 없애준다는 의미에서 Antigravity라는 이름이 붙은 것이 아닐까 생각됩니다.

일단 아래 링크에서 다운을 받아서 설치부터 하면 됩니다.

<Link url="https://antigravity.google/" />

## Antigravity의 특징

Antigravity는 단순히 코드를 대신 짜주는 것을 넘어서 디자이너와 QA(테스트) 담당자의 역할까지 수행하는 **올인원 에이전트를 지향**합니다. 특히 구글의 최신 기술력이 집약된 다음 기능들이 핵심입니다.

- **Nano Banana(나노 바나나) 모델 통합:**
  텍스트로 코딩만 하는 것이 아닙니다. 구글의 이미지 생성 특화 모델인 `Nano Banana`가 탑재되어 있어, 개발자가 "이런 느낌의 로그인 화면 디자인해줘"라고 하면 UI 목업(Mockup)을 즉시 생성합니다. 더 놀라운 점은 게임 캐릭터, 아이콘, 배경 이미지 같은 시각적 애셋(Asset)까지 직접 만들어내어, 디자이너 없이도 그럴듯한 결과물을 낼 수 있다는 점입니다.
- **Browser Control (브라우저 제어 및 자율 테스트):**
  기존 AI는 코드를 작성해주고 ‘실행은 인간아 니가 해봐라’라는 식이었습니다. 하지만 Antigravity의 에이전트는 **직접 크롬 브라우저를 열고, 클릭하고, 타이핑하며 테스트**합니다. 에이전트가 브라우저에서 작업 수행하며 오류가 발생하면 스스로 코드를 수정(Self-healing)하고 다시 테스트하는 놀라운 모습을 보여줍니다.
- **Multi-Agent Orchestration (멀티 에이전트 작업):**
  Antigravity에는 Manager View라는 모드가 따로 존재합니다. 여기서 유저가 마치 PM(프로젝트 매니저)처럼 여러 에이전트에게 동시에 작업을 지시할 수 있습니다. - "에이전트 A는 로그인 API를 짜고, 에이전트 B는 **Nano Banana** 모델로 로그인 페이지 디자인을 만들고, 에이전트 C는 문서(Readme)를 작성해."라고 명령하면 3명의 AI가 동시에(Asynchronous) 각자의 작업을 수행합니다.

## Antigravity로 웹사이트 만들어보기

Antigravity에게 일을 시키기 전에 Setting을 몇 가지 바꿨습니다. Agent 셋팅에서 표시된 부분을 Turbo로 바꿨습니다. Turbo로 바꾸면 Agent가 터미널 명령을 실행할 때 물어보지 않고 스스로 판단내려서 터미널 명령을 내립니다.

<Image alt="Agent 세팅" src="/google-agent-ide-antigravity/01.png" caption="Agent 세팅" />

브라우저 셋팅도 아래와 같이 설정해주었습니다. 이렇게 하면 Agent가 브라우저를 제어할 수 있습니다. 관련 프로그램을 설치하라고 나오면 설치하면 됩니다.

<Image alt="브라우저 세팅" src="/google-agent-ide-antigravity/02.png" caption="브라우저 세팅" />

한국에 상장된 해외투자 ETF를 조회할 수 있는 사이트를 만들어보려고 ETF 데이터를 수집해놨었습니다. 그래서 다음과 같은 프롬프트를 Antigravity에게 주고 웹사이트를 만들어보라고 시켜봤습니다.

```
Build a Next.js website that provides information on foreign investment ETFs listed in Korea. The data to display to users is contained in an etf_list.json file.

Key Requirements:

Functionality: Users must be able to filter by category and search by ETF name.

List View: Display only the ETF Name and Ticker Number. Clicking on an item should navigate to a detail page.

Design: The site must be mobile-optimized. I do not want a full-width fluid layout; instead, the content should be inside a fixed-width container centered in the browser window.
```

아래는 Antigravity에서 제공하는 Agent Manager 화면입니다. 위 프롬프트를 실행했을 때 상태입니다. 알아서 프로젝트를 세팅하고 어떤식으로 진행했는지 문서도 만들어서 보여줍니다. 읽어보고 Review에 수정 사항이나 다음 명령을 내리면 다음 작업을 시작합니다. 코드가 있는 에디터도 제공하는데 굳이 꼭 필요한 상황이 아니라면 굳이 열어볼 필요도 없었습니다.

파비콘도 만들어 달라고 하니까 뚝딱 만들어 줍니다. 웹사이트 만들고 파비콘 뭘로하지 고민 많이 했었는데 특별한 디자인이 필요한 게 아니라면 이렇게 요청하니 너무 편합니다.

이런 쉬운 과정을 몇 번 거치니 웹사이트가 하나 뚝딱 만들어졌습니다. 배포 작업, 광고 설정 등만 제가 하고 코드는 하나도 만지지 않았습니다. 디자인도 모던하니 나쁘지 않은 것 같습니다.

## 마무리

지금까지 구글의 Agent-first IDE, **Antigravity**에 대해 알아보았습니다. Antigravity와 같은 도구의 등장으로 이제는 **'무엇을 만들 것인가'** 하는 기획력과 창의력이 더 중요한 시대로 접어든 것 같습니다. 물론 **여전히 이런 도구를 더 잘 쓰려면 소프트웨어의 기본은 알고 있어야 한다는 생각이 바뀌지는 않았습니다.** 이제 기본기만 탑재하면 정말 빠르게 나아갈 수 있을 것 같습니다.
